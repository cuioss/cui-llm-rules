# Plan Persistence Standards

## Directory Structure

### Location

**Primary Location**: `.claude/plans/`

### Organization

Each plan uses a dedicated subdirectory:

```
.claude/plans/
  └── {task-name}/
      ├── plan.md                        # Tasks only (phases, tasks, status)
      ├── config.md                      # Configuration from init phase
      ├── references.md                  # Reference management and links
      └── implementation-requirements.md # Runtime artifact (generated by refine phase)
```

## Naming Conventions

### Directory Naming

**Task Name Rules**:
- Lowercase with hyphens (kebab-case)
- Descriptive of task/feature
- Can include issue numbers
- Maximum 50 characters
- Must be valid directory name (no special characters except hyphen)
- No spaces (use hyphens instead)

**Valid Examples**:
- `jwt-auth`
- `issue-123`
- `refactor-auth-service`
- `add-user-permissions`

**Invalid Examples**:
- `JWT Auth` (has space)
- `add_user_permissions` (use hyphens, not underscores)
- `refactor/auth/service` (no slashes)

### File Names (Fixed)

- `plan.md` - Tasks only (phases, tasks, status)
- `config.md` - Configuration from init phase
- `references.md` - References and links
- `implementation-requirements.md` - Runtime artifact (generated by refine phase)

## File Formats

### plan.md Structure

```markdown
# Task Plan: {task_title}

**Configuration**: See [config.md](./config.md)
**References**: See [references.md](./references.md)

**Current Phase**: {init|refine|implement|verify|finalize}
**Current Task**: {task-id or "none"}

---

## Phase Progress

| Phase | Status | Tasks | Completed |
|-------|--------|-------|-----------|
| init | {status} | {count} | {completed}/{count} |
| refine | {status} | {count} | {completed}/{count} |
| implement | {status} | {count} | {completed}/{count} |
| verify | {status} | {count} | {completed}/{count} |
| finalize | {status} | {count} | {completed}/{count} |

---

## Phase: {phase} ({status})

### Task N: {task-name}

**Phase**: {phase}
**Goal**: {goal}

**Acceptance Criteria**:
- {criterion-1}
- {criterion-2}

**Checklist**:
- [ ] {item-1}
- [ ] {item-2}

---

## Completion Criteria

All phases must be completed and all tasks marked with `[x]` before plan is complete.
```

## Reference Types

### 1. Issue and Branch

Track issue implementation and link to branches.

### 2. Local Files

Categories:
- Implementation Files: Source code
- Configuration Files: Config files
- Test Files: Test source files

### 3. ADRs (Architectural Decision Records)

Use `adr-management` skill to:
- Verify ADR exists
- Create new ADRs
- Read ADR content

### 4. Interface Specifications

Use `interface-management` skill to:
- Verify interface exists
- Create new interfaces
- Read interface content

### 5. External Documentation

- Standards and Specifications (RFCs, W3C, ISO)
- Library Documentation
- Best Practices Guides

### 6. Dependencies

- Maven/npm dependencies (with versions)
- Related plans (prerequisites, dependencies)

## Version Control

**Recommendation**: Git-track plan directories

**.gitignore Consideration**:
```
# Track plans
!.claude/plans/

# Optionally ignore temp files
.claude/plans/*/temp-*.md
```

**Commit Message Format**:
```
docs(plan): update jwt-auth plan with phase 2 tasks
```
