# Plan Files Architecture

## Skill Separation

```
Phase Skills                    Persistence Skill
┌─────────────┐                ┌──────────────┐
│ plan-init   │──────────────▶│              │
├─────────────┤                │              │
│ plan-refine │──────────────▶│  plan-files  │──▶ .cui/plans/
├─────────────┤                │              │
│ plan-impl   │──────────────▶│              │
├─────────────┤                │              │
│ plan-verify │──────────────▶│              │
├─────────────┤                │              │
│ plan-final  │──────────────▶└──────────────┘
└─────────────┘
```

**Separation of Concerns**:
- Phase skills handle business logic (what to do)
- plan-files handles file I/O (how to persist)

## Operations Summary

| Operation | Purpose | Tool |
|-----------|---------|------|
| create-directory | Create plan directory | Bash |
| read-plan | Read plan.md | Read |
| read-config | Read config.md | Read |
| get-references | Read references.md | Read |
| write-plan | Create/update plan.md | Write/Edit |
| write-config | Create config.md | Write |
| write-references | Create/update references.md | Write/Edit |
| update-progress | Update task/phase status | Edit |

## File Structure

```
.cui/plans/{task-name}/
├── plan.md           # Task plan with phases and checklists
├── config.md         # Configuration (build, workflow, context)
├── references.md     # References (issue, ADRs, interfaces, files)
└── implementation-requirements.md  # Generated by refine phase
```

## Validation Rules

### Plan Validation
- Required headers present
- Phase Progress Table exists
- All tasks have status indicators
- All tasks have acceptance criteria
- Phase statuses are consistent

### Config Validation
- All required fields present
- Field values are valid
- Branch and issue format correct

### References Validation
- Issue and branch present
- ADR references valid (via adr-management skill)
- Interface references valid (via interface-management skill)
- File paths valid

## Integration Notes

### ADR Verification

Before adding ADR reference, verify via adr-management skill:
```
Skill: cui-documentation-standards:adr-management
operation: read
adr_identifier: {ADR-NNN}
```

### Interface Verification

Before adding interface reference, verify via interface-management skill:
```
Skill: cui-documentation-standards:interface-management
operation: read
interface_identifier: {IF-NNN}
```
