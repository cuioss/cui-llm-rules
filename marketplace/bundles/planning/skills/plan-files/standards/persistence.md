# Plan Persistence Standards

## Directory Structure

### Location

**Primary Location**: `.plan/plans/`

### Organization

Each plan uses a dedicated subdirectory:

```
.plan/plans/
  └── {task-name}/
      ├── plan.md                        # Tasks only (phases, tasks, status)
      ├── config.toon                    # Configuration (TOON format)
      ├── references.toon                # Reference management (TOON format)
      └── implementation-requirements.md # Runtime artifact (generated by refine phase)
```

## Naming Conventions

### Directory Naming

**Task Name Rules**:
- Lowercase with hyphens (kebab-case)
- Descriptive of task/feature
- Can include issue numbers
- Maximum 50 characters
- Must be valid directory name (no special characters except hyphen)
- No spaces (use hyphens instead)

**Valid Examples**:
- `jwt-auth`
- `issue-123`
- `refactor-auth-service`
- `add-user-permissions`

**Invalid Examples**:
- `JWT Auth` (has space)
- `add_user_permissions` (use hyphens, not underscores)
- `refactor/auth/service` (no slashes)

### File Names (Fixed)

- `plan.md` - Tasks only (phases, tasks, status)
- `config.toon` - Configuration (TOON format)
- `references.toon` - References and links (TOON format)
- `implementation-requirements.md` - Runtime artifact (generated by refine phase)

## File Formats

### plan.md Structure

```markdown
# Task Plan: {task_title}

**Configuration**: See [config.toon](./config.toon)
**References**: See [references.toon](./references.toon)

**Current Phase**: {init|refine|implement|verify|finalize}
**Current Task**: {task-id or "none"}

---

## Phase Progress

| Phase | Status | Tasks | Completed |
|-------|--------|-------|-----------|
| init | {status} | {count} | {completed}/{count} |
| refine | {status} | {count} | {completed}/{count} |
| implement | {status} | {count} | {completed}/{count} |
| verify | {status} | {count} | {completed}/{count} |
| finalize | {status} | {count} | {completed}/{count} |

---

## Phase: {phase} ({status})

### Task N: {task-name}

**Phase**: {phase}
**Goal**: {goal}

**Acceptance Criteria**:
- {criterion-1}
- {criterion-2}

**Checklist**:
- [ ] {item-1}
- [ ] {item-2}

---

## Completion Criteria

All phases must be completed and all tasks marked with `[x]` before plan is complete.
```

### config.toon Structure (TOON format)

```toon
# Plan Configuration

plan_type: {implementation|simple|plugin-development}
branch: {branch-name}
issue: {issue-reference or "none"}

technology: {java|javascript|mixed|none}
build_system: {maven|gradle|npm|npx|none}

compatibility: {breaking|deprecations}
commit_strategy: {fine-granular|phase-specific|complete}
finalizing: {commit-only|pr-workflow}
```

## Reference Types

### 1. Issue and Branch

Track issue implementation and link to branches.

### 2. Local Files

Categories:
- Implementation Files: Source code
- Configuration Files: Config files
- Test Files: Test source files

### 3. ADRs (Architectural Decision Records)

Use `adr-management` skill to:
- Verify ADR exists
- Create new ADRs
- Read ADR content

### 4. Interface Specifications

Use `interface-management` skill to:
- Verify interface exists
- Create new interfaces
- Read interface content

### 5. External Documentation

- Standards and Specifications (RFCs, W3C, ISO)
- Library Documentation
- Best Practices Guides

### 6. Dependencies

- Maven/npm dependencies (with versions)
- Related plans (prerequisites, dependencies)

## Version Control

**Plans are NOT tracked in git** - they are working session artifacts excluded by `.gitignore`.

The `.plan/*` pattern in `.gitignore` excludes plan directories. This is intentional:
- Plans are session-specific working documents
- Plan progress updates would create noise in git history
- Implementation code (not plans) is what gets committed
- Plans can be cleaned up after completion via `/plan-manage action=cleanup`

**What IS committed**:
- Source code changes made during implement phase
- Test files created during implement phase
- Documentation updates (ADRs, interfaces, READMEs)

**What is NOT committed**:
- `.plan/plans/` directories and contents
- Plan progress tracking (plan.md, config.toon, references.toon)
