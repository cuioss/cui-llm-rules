# Triage Extension Contract

Contract for domain-specific triage extension skills.

## Purpose

Triage extensions provide domain-specific guidance during plan finalization. They help Claude determine how to handle findings (fix, suppress, accept) based on domain-specific criteria.

## Registration

Register a triage extension in the domain manifest:

```json
{
  "extensions": {
    "triage": "pm-dev-java:java-triage"
  }
}
```

## Required Structure

### Frontmatter

The extension skill MUST include:

```yaml
---
name: {domain}-triage
description: Triage extension for {Domain} findings
implements: plan-marshall:domain-extension-api/triage-extension
---
```

### Required Sections

| Section | Required | Purpose |
|---------|----------|---------|
| `## Suppression Syntax` | Yes | How to suppress findings in code |
| `## Severity Guidelines` | Yes | Fix/suppress/accept criteria |
| `## Acceptable to Accept` | Yes | When accepting is appropriate |

## Section Content

### Suppression Syntax (Required)

Document how to suppress findings in this domain:

```markdown
## Suppression Syntax

### SonarQube Suppressions
```java
@SuppressWarnings("java:S1234")  // Single rule
@SuppressWarnings({"java:S1234", "java:S5678"})  // Multiple rules
```

### SpotBugs Suppressions
```java
@SuppressFBWarnings("NP_NULL_ON_SOME_PATH")
```

### IDE Suppressions
```java
//noinspection unchecked
```
```

### Severity Guidelines (Required)

Define criteria for handling different severity levels:

```markdown
## Severity Guidelines

| Severity | Default Action | Override Conditions |
|----------|---------------|---------------------|
| BLOCKER | Fix | Never suppress unless false positive |
| CRITICAL | Fix | Suppress only with documented justification |
| MAJOR | Fix if simple | May suppress for legacy code |
| MINOR | Fix if trivial | May accept for stylistic preferences |
| INFO | Accept | Fix if improving readability |
```

### Acceptable to Accept (Required)

Define when accepting (not fixing, not suppressing) is appropriate:

```markdown
## Acceptable to Accept

Accept findings without action when:

### Generated Code
- Code generated by annotation processors
- Code generated by build tools
- Generated API clients

### Test Code Patterns
- Mock setups that appear as unused assignments
- Test data that triggers naming conventions
- Assertions that appear as dead code

### Framework Requirements
- Required public constructors for CDI
- Required no-args constructors for JPA
```

## Validation

Validation checks:

1. **Frontmatter**: `implements` field references this contract
2. **Required sections**: All three required sections exist
3. **Section content**: Required sections are not empty

## Example

```markdown
---
name: java-triage
description: Triage extension for Java findings
implements: plan-marshall:domain-extension-api/triage-extension
---

# Java Triage Extension

## Suppression Syntax

### SonarQube
```java
@SuppressWarnings("java:S1234")
```

### SpotBugs
```java
@SuppressFBWarnings(value = "NP_NULL_ON_SOME_PATH", justification = "Null check in caller")
```

### Checkstyle
```java
// CHECKSTYLE:OFF: LineLength
long line here
// CHECKSTYLE:ON: LineLength
```

## Severity Guidelines

| Severity | Action | Notes |
|----------|--------|-------|
| BLOCKER | Fix | Security issues, critical bugs |
| CRITICAL | Fix | Data integrity, resource leaks |
| MAJOR | Fix/Suppress | Code smells, complexity |
| MINOR | Accept/Fix | Naming, formatting |
| INFO | Accept | Suggestions |

## Acceptable to Accept

### Generated Code
- Lombok-generated methods
- MapStruct implementations
- JAXB/Jackson annotations

### Test Patterns
- `@Disabled` tests with tracked issues
- Parameterized test complexity
- Mock verification patterns

### Framework Conventions
- CDI `@Inject` on private fields
- JPA entity requirements
- JAX-RS annotations
```

## Integration

During plan finalization:

1. Load domain triage extension if configured
2. Apply severity guidelines to categorize findings
3. Use suppression syntax for suppressions
4. Apply "acceptable to accept" criteria
