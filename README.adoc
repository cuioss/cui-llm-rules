:toc: left

Claude Code marketplace bundles providing comprehensive development standards and automation for CUI OSS projects.

== Installation

=== Install All Bundles

Install the complete marketplace (recommended):

[source,bash]
----
git clone https://github.com/cuioss/cui-llm-rules.git
----

For Claude Code, add to your global or project settings:

[source,json]
----
{
  "marketplaces": [
    {
      "path": "/path/to/cui-llm-rules/marketplace"
    }
  ]
}
----

=== Install Individual Bundles

Install specific bundles via Claude Code (requires marketplace setup):

[source,bash]
----
/plugin install cui-java-expert
/plugin install cui-frontend-expert
/plugin install cui-maven
----

== Architecture

=== Bundle Structure

Each bundle is self-contained with:

----
bundle-name/
├── .claude-plugin/
│   └── plugin.json          # Bundle manifest
├── agents/                  # Specialized task agents
│   └── *.md
├── commands/                # Slash commands
│   └── *.md
├── skills/                  # Development standards
│   └── skill-name/
│       ├── SKILL.md
│       ├── standards/
│       └── README.md
└── README.md               # Bundle documentation
----

=== Component Types

**Skills**::
Development standards and best practices loaded by AI assistants. Skills provide expert knowledge in specific domains (Java, JavaScript, testing, documentation, etc.).

**Agents**::
Specialized autonomous agents that handle focused tasks like build execution, coverage analysis, or documentation validation. Agents use specific tool permissions and follow architectural patterns.

**Commands**::
Interactive slash commands (`/command-name`) that users invoke directly. Commands orchestrate agents and workflows to accomplish complex multi-step tasks.
